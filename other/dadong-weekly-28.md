2. 多语言有用，对于总体流量有帮助；
3. 外链重要，外链很重要，外链超级重要；
4. 多上手多上站，光看不练假把式。做出成绩的群友都至少是做了四五个站以后才找到感觉；
5. 找新词能让你更早得到正反馈；
6. 流量突然大增，谷歌Adsense可能风控你。可以用cloud flare来block一些AI爬虫程序，和一些恶意IP。但是只要你的流量是真实的，迟早会取消你的风控；
7. 从谷歌的政策来说，一个人只能有一个adsense账号。账号养久了，广告单价能高一些。一个账号能用在多个网站上；
8. 在发外链的过程中，你能发现其他一些网站也在通过外链推广，那你也可以去评估这个需求，是不是可以来做同样的需求；
9. 长尾持的竞争少，但是流量不一定少；
10. 新词KD低，竞争比较低，但是要快；
11. Generator这个词根比较好用；
12. 需求可以交叉验证，比如谷歌搜索谷歌下拉reddit，x,tk；
13. 一个页面做一个关键词，不要很多页面做一个关键词，也不能一个页面做很多关键词；
14. 流量流量流量聚焦流量；
15. 不要只是闷头做产品，要去推广，有预算的时候要舍得花钱；
16. bing也不能忽视；
17. 别人做了你也能做，不要因为别人做了，就觉得自己不能做；
18. 不断上内页，不断上外链；
19. 第一个版本可以制作单页快速迭代，MVP；

### 杨书记

![](https://cdn.mundane.ink/20260112220011011.png)

### laughing_man

听了哥飞 2025 年度 Top 30 的复盘直播，总时长长达6小时，都是最一线的干货和实操经验。强烈推荐大家去听。 

我把其中反直觉、且带有实战案例的认知，和ai一起梳理了一下。以下是核心复盘：

1. 冠军的逻辑：不是找蓝海，而是修补破烂 大部分人都在疯狂找没人做的新词，但李太白（第1名） 做的是老词。 Case： 他发现霸榜的前几名往往是大厂年久失修的内页，或者满屏广告的老站。 洞察： 竞品做得烂，就是最大的蓝海。不需要颠覆式创新，只需要把 UX 做得流畅、无 Bug。Google 的算法非常智能，一旦用户在你这里停留时间变长，排名迅速替换。
2. 唯快不破：甚至不需要开发 赚钱的是“需求”，不是“代码”。 Case - Zero（第2名）： 上线第一天 PV 100万。根本没写复杂后端，直接用静态页面承接爆发流量。 Case - 圈圈（第12名）： 像素级复刻。不自己想功能，盯着行业第一名（硬核玩家），对方有什么，马上复刻一个更好用的。 Case - 梦见（月度冠军）： 先上 Waitlist（假门测试），测出有流量了，再连夜写代码去兑现。 洞察： 发现新词，3小时内必须上线占位。等你想好架构，红利期（通常仅2周）早就结束了。 
3. 流量诅咒：没有 Serverless 就是自杀 如果你还在用按量付费的云服务器，根本玩不起这种“一波流”游戏。 Case： Zero（第2名） 上线即爆（日 PV 100万），结果 AWS 账单爆炸。如果没及时迁到 Cloudflare，赚的广告费还不够付服务器钱。 技术栈共识： Cloudflare + Vercel + Supabase 或纯静态 HTML。 洞察： 做流量站，必须把边际成本降为 0。 
4. 决定贫富的不是 UV，是含金量 赛道和地区，决定了你的努力值多少钱。 Case - Alan（第16名）： 60万 UV（西语区） -> 收入 $1000+。 Case - 默默Z楼（第10名）： 56万 UV（欧美区 AI工具） -> 收入 $10000+。 Case - 范文（第8名）： 62万 UV（游戏站） -> 流量大但单价低。 洞察： 选词时，别只看搜索量（Volume），更要看 CPC（点击单价）。做小语种容易拿流量，但变现时会让你怀疑人生。 
5. 外链的工业化 获奖者没有一个是“佛系”等收录的，全是“暴徒”。 Case - 范文（第8名）： 只要一上线，立刻利用工具/插件，自动去几百个导航站、论坛暴力群发。 Case - 默默Z楼（第10名）： 甚至专门雇佣实习生，每天只做这一件事：机械式地发外链。 洞察： 新站 Sandbox 期，必须通过大量外部链接告诉爬虫“我在这里”。 
6. 程序员的困境：死于过度工程化 在 AI 能写代码的时代，审美、交互设计、SEO 语感才是核心壁垒。 Case - Winter（设计师）： 不懂后端，但把网站做得极美，交互极好。用户进来就不想走，高留存带来高排名。 Case - 李太白（投手出身）： 不懂代码，但他懂转化率，知道按钮放哪用户最想点。 反面教材： 很多程序员花了大量时间重构代码、优化性能，结果网站根本没人看。 
7. 极简实操 SOP 找词：只盯 Google Trends 飙升词，或用 Semrush 查竞品漏网之鱼。 验证：搜一下前三名，如果看到上世纪 90 年代风格的网页，直接入场。 建站：静态 HTML + AI 生成，成本控制在域名费（$10)以内。 闭环：有流量进来了才开始精修 UI 和功能，没流量直接换词。 
8. 在这个时代，赚钱的往往不是写代码最厉害的人，而是那个懂设计、懂人性、且执行力极强的超级个体。

### 我自己的感受

我自己听下来的感受，我感觉他们有一个共同的点。就是先找需求，做一个mvp出来，加一些外链。然后继续寻找需求。直到发现做过的网站中有流量起来了，然后集中精力做那个有流量的网站。

我发现好几个人的做法是这样的。毕竟人的精力和热情都是有限的，不可能把每一个网站做的面面俱到，也不可能一直去迭代一个没有正反馈的网站。

关键是做出来的东西要有人用，这个的关键是找到比较新的或者竞争的人比较少的需求，然后做一个mvp出来。这样的话挖掘需求就很关键了，平时要多刷刷hack news、product hunt、reddit什么的去发现需求。

当然，外链也很关键，前面几个总结都提到了。最好是有个人专门发外链，而自己专注于更重要的事。
